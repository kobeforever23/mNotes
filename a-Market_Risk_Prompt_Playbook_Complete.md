## Prompt Quality Framework

### Best Practices Embedded

Every prompt in this collection follows:

1. **Clear Role Setting** - Defines the AI's persona and expertise
2. **Data Integrity Rules** - Specifies allowed data sources
3. **Structured Output** - Explicit format requirements
4. **Validation Requests** - Asks for confidence levels, caveats
5. **Guardrails** - Prevents hallucination, ensures accuracy

### Quality Indicators

- ✅ Includes context and background
- ✅ Specifies output format explicitly
- ✅ Has constraints and guardrails
- ✅ Requests examples or evidence
- ✅ Defines role/persona clearly

## Quick Reference

### Prompt Anatomy

```
[ROLE] You are a [specific expertise] in [context]

[CONSTRAINTS] Follow these rules:
- Use ONLY [specified data sources]
- Do NOT [prohibited actions]

[TASK] Analyze/Create/Explain [specific request]

[FORMAT] Output as:
1. Executive Summary
2. Detailed Analysis
3. Recommendations

[VALIDATION] Include confidence levels and caveats
```

### Power Modifiers

Add these to enhance any prompt:

- "Think step-by-step before answering"
- "Flag any data quality concerns"
- "Include confidence levels for each finding"
- "Highlight areas requiring human judgment"
- "Note any assumptions made"
